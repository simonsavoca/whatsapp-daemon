API HTTP interne du daemon, servie en local sur 127.0.0.1:3099 (non exposée à l'extérieur). Consommée par les applications clientes du daemon et par ce dashboard.

Authentification

GET /auth/status

État de la connexion WhatsApp et du compte lié.

Réponse

{ "connectionState": "open", "user": { "name": "...", "id": "..." }, "messageCount": 123 }
GET /auth/qr

Dernier QR code d'authentification (data URL PNG), ou null si déjà connecté ou pas encore généré.

Réponse

{ "qr": "data:image/png;base64,...", "connectionState": "connecting" }
POST /auth/reset

Destructif — supprime la session WhatsApp en cours et redémarre la connexion pour générer un nouveau QR à scanner.

Réponse

{ "ok": true }

Envoi & groupes

POST /send

Visible sur WhatsApp mobile — envoie un message à un contact/groupe recherché par nom (sous-chaîne, insensible à la casse).

Corps

{ "query": "Virginie", "message": "Coucou" }

Réponse

{ "ok": true, "msg": "Message envoyé à \"Virginie\"", "jid": "..." }
// 409 si plusieurs correspondances : { "error": "...", "matches": [{ "name", "jid", "isGroup" }] }
POST /join-group

Rejoint un groupe WhatsApp via un lien d'invitation.

Corps

{ "inviteLink": "https://chat.whatsapp.com/CODE" }

Réponse

{ "ok": true, "jid": "..." }

Messages

GET /messages/recent?limit&filter

Derniers messages capturés par le daemon. limit (défaut 20) et filter (nom du chat ou texte) optionnels.

Réponse

{ "messages": [ { "id", "chat_name", "text", "timestamp", "read_at", ... } ] }
GET /messages/unread

Messages non lus (basé sur read_at en base).

Réponse

{ "messages": [ ... ] }
POST /messages/read

Irréversible — marque des messages comme lus (ticks bleus, visible sur tous les appareils). Passer ids pour ne marquer que des messages précis, sinon upTo marque tout jusqu'à ce timestamp (défaut : maintenant).

Corps

{ "ids": [12, 13] }  // ou : { "upTo": "2026-07-05T10:00:00.000Z" }

Réponse

{ "ok": true, "count": 2 }

Base de données

GET /db/chats?filter

Table chats, triée par dernière mise à jour. filter optionnel (nom).

Réponse

{ "chats": [ { "jid", "name", "is_group", "updated_at" } ] }
GET /db/messages?limit&offset&filter

Table messages paginée (limit défaut 50, offset défaut 0). filter optionnel (nom du chat ou texte).

Réponse

{ "messages": [ ... ], "total": 431 }
POST /db/reset

Destructif — vide les tables messages et chats (le fichier whatsapp.db et son schéma sont conservés). N'affecte pas la session WhatsApp authentifiée.

Réponse

{ "ok": true }
